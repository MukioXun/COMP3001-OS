1. 虚拟内存概念与优点

1.1 什么是虚拟内存 必考

虚拟内存：使应用程序认为它拥有一个连续的、完整的大地址空间，而实际上程序被分散在物理内存碎片和磁盘上，在需要时进行数据交换。

实现技术：段页管理 + 部分加载 + 按需调页 + 换入换出

效果：每个进程独享 CPU 可寻址的整个线性地址空间（32位机4GB），可远大于物理内存。

基础：程序局部性原理（80/20原则：80%的时间访问20%的地址），使得只需将当前用到的页保留在内存中。

1.2 虚拟内存的优点 重点

  • 优点1：地址空间 > 物理内存，用户可编写比物理内存更大的程序，简化编程
  • 优点2：内存中可容纳更多进程（并发度高），利用率高；未使用的页（如异常处理代码）从不调入内存
  • 优点3：程序启动更快（只需加载初始部分即可开始执行）

2. 按需调页（请求调页）

2.1 从交换到调页的演化 重点

  • 早期：整个程序必须装入内存才能执行，内存不足则无法运行
  • 进程级交换：内存不足时以进程为单位换出，颗粒度太大
  • 调页（惰性交换）：以页为单位换入换出
  • 请求调页（按需调页）：只有进程真正访问某页时才将其调入，未访问的页可能直到程序结束也不调入

2.2 改造后的页表项 必考

页表项增加有效/无效位（Present位，P位）：

  • P=1：页在物理内存中，页表项存储物理页框号
  • P=0：页不在内存中，页表项存储磁盘位置信息（如swap分区的扇区号）

其他重要位：访问位A（MMU在读/写时置1，软件清0）、修改位D（Dirty）（MMU在写时置1，换出时决定是否写回磁盘）

易错：页换出后，原来存放页框号的字段改为存储磁盘地址，P位清0，不会清除该字段。

2.3 缺页中断处理流程 必考

  1. CPU发出访存指令，MMU查页表，发现P位=0
  2. MMU产生缺页中断（Page Fault），CPU转入缺页中断处理程序
  3. 中断处理程序从磁盘读取缺页内容
  4. 在物理内存中选一个空闲页框（若无空闲则先置换）
  5. 将磁盘上的页数据读入该页框，修改页表项（P位=1，填入页框号）
  6. 重新执行引起缺页的指令（需硬件支持保存EPC）
易错：缺页处理完成后必须重新执行（而不是继续执行）引发缺页的那条指令。

3. 页面置换算法

3.1 FIFO（先进先出） 必考

淘汰最早调入内存的页面。实现简单，但性能不佳。

示例（3帧，引用序列 A B C A B D A D B C B）：FIFO 产生 7次缺页

缺点：可能淘汰频繁使用的页（"不合适"）；存在 Belady异常（增加帧数缺页反而增多）

3.2 OPT（最优页面置换） 必考

淘汰未来最远时才会使用的页。理论最优，缺页数最少，由 Belady 于1966年提出。

示例（同上）：OPT 产生 5次缺页（< FIFO的7次）

缺点：需要预知未来访问序列，实际不可实现，仅作为评价基准。

用途：用过去的历史预测未来 --> 启发 LRU 算法。

3.3 LRU（最近最少使用） 必考

淘汰最近最长时间未使用的页（OPT的可实现近似，用过去预测未来）。

示例（同上）：LRU 产生 5次缺页（= OPT）

准确实现：

  • 计数器法：每次页引用时全局计数器加1，将值写入页表项。置换时选计数最小的页。代价大（每次访问需修改时间戳，需找最小值），几乎不用。
  • 页码栈法：维护页码栈，每次引用将页号移到栈顶，栈底即为LRU页，直接替换。代价仍大，实际使用少。

LRU 属于栈式算法，满足 M(m,r) ⊆ M(m+1,r)，不会出现 Belady 异常。

3.4 近似LRU：时钟算法（Clock/NRU） 必考

用引用位（Reference bit）代替精确时间戳，将"何时使用"简化为"是否最近使用过"。

简单Clock算法（二次机会算法，SCR）：

  • 所有帧组成循环队列，每帧维护引用位 R
  • 访问某页时，硬件自动将 R 置1
  • 缺页时：扫描循环队列，R=1则清0继续扫描；R=0则淘汰该页，指针移到下一位置

示例（同上）：Clock 产生 7次缺页（= FIFO，因为样本中所有页引用位都被置1后退化为FIFO）

实际上Clock是公认的很好的近似LRU算法，在真实工作负载下性能接近LRU。

改进型Clock（双指针）：指针1选择淘汰页（慢），指针2定时清除R位（快）；避免记录太长历史信息退化为FIFO。

改进型Clock（4状态）：结合访问位A和修改位M，优先淘汰A=0,M=0的页（未访问未修改，换出代价最低）。

3.5 置换算法对比 重点

算法策略缺页数（示例3帧）可实现Belady异常
FIFO淘汰最早调入的页7是有
OPT淘汰未来最远使用的页5（最少）否无（栈式）
LRU淘汰最近最长未用的页5代价大无（栈式）
Clock/NRU近似LRU（引用位）接近LRU是（高效）通常无

4. 其他相关问题

4.1 写时复制（Copy-on-Write, CoW） 重点

fork() 创建子进程时，父子进程共享同一份物理页，页表项标记为只读。

当任意进程尝试写入共享页时，触发保护故障，内核复制该页，两进程各自拥有独立副本，再完成写操作。

优点：延迟拷贝（可能永远不需要复制），加速进程创建。

4.2 交换空间（Swap Area） 常考

换出的页面存储在磁盘的交换区：

  • 基于文件系统：Windows的 pagefile.sys
  • 独立磁盘分区（RAW）：Linux的 swap partition（不需要文件系统结构）

4.3 工作集与颠簸 必考

工作集（Working Set）：进程最近N次内存引用中访问的不同页面集合，是程序局部性的近似。

工作集分配策略：固定分配（平均/比例/优先级）；可变分配（根据缺页率动态调整，性能更好）。

颠簸（Thrashing）：多个进程对物理内存的总需求超过可用量，导致频繁缺页置换，CPU利用率骤降，进程不断等待调页。

颠簸成因：CPU利用率低 --> OS增加进程 --> 缺页率升高 --> CPU利用率更低 --> 恶性循环。

防止颠簸：给每个进程分配足够多的帧（基于工作集模型或PFF页错误率）；必要时换出部分进程。

易错：所有置换策略都不能完全避免颠簸，颠簸是系统级问题，根本原因是总需求超过总供给。

4.4 Belady异常 常考

对某些页面置换算法（如FIFO），增加分配的帧数，缺页次数反而增加的异常现象。

经典例子：引用序列 1,2,3,4,1,2,5,1,2,3,4,5；FIFO 3帧9次缺页，4帧10次缺页。

栈式算法（满足 M(m,r) ⊆ M(m+1,r)）不会出现 Belady 异常，LRU 和 OPT 是栈式算法；FIFO 不是。

4.5 全局置换与局部置换 重点

  • 局部置换：进程只能淘汰自己的页面，保护性好、公平，但单进程缺页不能从全局获益
  • 全局置换：实现简单，可以淘汰任何进程的页面；但一个频繁缺页的进程会疯狂置换其他进程的页面，不公平

4.6 虚拟内存中的程序优化 了解

  • 代码优化：使热点代码（80%执行时间对应的20%代码）紧凑、集中，减少跨页访问
  • 数据优化：将会一起访问的数据放在同一页或相邻页（如链表节点就近分配），减少缺页次数

5. CSAPP：Cache与虚拟地址翻译

5.1 高速缓存（Cache）组织 重点

Cache参数：(S, E, B) -- S=组数(2^s)，E=每组行数，B=块大小(2^b字节)，容量 C=S×E×B

物理地址分解：[标记(t位) | 组索引(s位) | 块偏移(b位)]

查找过程：① 用s位选组 --> ② 将t位与组内所有行的标记比较 --> ③ 有效位=1且标记匹配则命中，取偏移处数据

映射方式特点适用
直接映射（E=1）速度快，冲突率高，不需要替换算法大容量Cache
组相联（E=2,4...）速度折中，冲突率低，需要替换算法（LRU等）各种容量
全相联（E=所有行）冲突率=0，硬件复杂，成本高小容量（如TLB）

Intel Core i7 L1 Cache：32KB，8路组相联（E=8），64字节/块（b=6），64组（s=6）

5.2 TLB组织与虚拟地址翻译 必考

TLB地址字段：虚拟地址拆分为 [VPN(虚拟页号) | VPO(虚拟页偏移)]

VPN进一步拆分为 [TLBT(TLB标记) | TLBI(TLB索引)] 用于查找TLB

TLB通常采用全相联或组相联，容量64~1024条（TLB对冲突容忍度极低，宁增延迟也要避免未命中）

完整翻译流程（TLB命中）：VA --> VPN/VPO拆分 --> TLB查找命中 --> 得PPN --> 物理地址=PPN+PPO --> 查L1 Cache

TLB未命中时：额外访问内存中的页表 --> 将PTE装入TLB --> 继续

5.3 Core i7 四级页表 常考

48位虚拟地址格式：[VPN1(9) | VPN2(9) | VPN3(9) | VPN4(9) | VPO(12)]

CR3 --> L1页表(全局目录) --> L2页表(上层目录) --> L3页表(中层目录) --> L4页表 --> 物理页

每个条目覆盖范围：L1=512GB，L2=1GB，L3=2MB，L4=4KB

PTE关键字段：P（存在位），R/W（读写权限），U/S（用户/内核），A（访问位），D（修改位，仅L4），XD（禁止执行）

5.4 Linux虚拟内存管理 常考

VMA（vm_area_struct）：描述进程虚拟地址空间中的一个连续区域，包含 vm_start/vm_end/vm_prot/vm_flags 等字段。

内存映射（mmap）：在进程虚拟地址空间与某个对象之间建立关联：

  • 普通文件：.text/.data 段，按需调页
  • 匿名文件：堆、栈、.bss段，初始全零，第一次引用时分配全零物理页

fork() 的内存映射：父子进程共享所有页（标记为CoW只读），写时才复制，加速创建。

execve() 的步骤：删除旧用户区域 --> 创建新VMA（代码/数据映射到目标文件，bss/栈映射到匿名文件）--> 设置PC指向入口点。
