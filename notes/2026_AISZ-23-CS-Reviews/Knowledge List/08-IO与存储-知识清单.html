1. I/O系统概述

1.1 I/O设备基本概念 重点

I/O设备三要素：端口（设备的I/O地址集）、总线（多部件分时复用的公共通信通道）、设备控制器（操作端口/总线/设备的电子器件集合）。

设备控制器内含数据寄存器、地址寄存器、控制寄存器、状态寄存器。OS通过读写控制器寄存器来操控设备。

设备寄存器编址方式：

  • 独立编址：需专用 I/O 指令（x86: in/out），如 out 0x21, AL
  • 内存映像编址：寄存器占用物理内存地址空间，使用普通 mov 指令，如 mov [0x8000f000], AL

1.2 I/O系统分层结构 重点

从上到下四层：

  1. 用户库函数接口：高级语言I/O库（printf/scanf等）
  2. I/O内核子系统：系统调用接口（open/read/write/close/ioctl），负责调度、缓冲、错误处理、保护
  3. 设备驱动程序（含中断服务程序）：隐藏设备差异，提供统一接口，读写控制寄存器
  4. I/O硬件：字符设备（鼠标/键盘）、块设备（磁盘）

OS为用户提供统一接口：不论何种设备，都通过 open/read/write/close 操作（设备即文件）。

2. I/O控制方式

2.1 四种控制方式概述 必考

方案原理CPU利用率特点
程序直接控制（轮询）CPU不断查询设备状态寄存器极低（忙等待）实现简单，CPU被独占
中断控制设备就绪后向CPU发中断信号较高（CPU可做其他事）大部分I/O使用此方式
DMA（直接内存存取）DMA控制器负责数据搬运，完成后中断CPU高（仅开始/结束时参与）需初始化DMA寄存器
通道控制通道有简单CPU功能，可执行通道程序最高（可并行）可控制多台设备，大型机使用

2.2 轮询方式（程序直接控制） 重点

步骤：CPU发 I/O 命令 --> 不断循环读状态寄存器 --> 设备就绪后读数据 --> 写内存。

缺点：CPU比外设快太多，轮询期间CPU无法做其他工作，浪费算力。

2.3 中断控制方式 必考

步骤：进程向控制器发 Read 命令 --> 进程阻塞，CPU做其他工作 --> 设备完成后发中断 --> 中断处理程序将数据从控制器传入内存 --> 唤醒等待进程。

易错：中断方式每传送一个字节都要发一次中断，若数据量大（如读磁盘），中断频率太高，CPU开销仍大。

2.4 DMA方式 必考

DMA（Direct Memory Access）初始化参数：内存起始地址、传送字节数、控制字（含启动位）写入DMA控制器寄存器。

工作过程：CPU初始化DMA --> 进程阻塞 --> DMA接管数据总线，独立完成整批数据传输（CPU工作周期挪用）--> 完成后 DMA 向 CPU 发中断 --> 中断处理程序唤醒进程。

DMA vs 中断：中断每字节一次，DMA每批数据一次；DMA适合大批量块设备传输。

2.5 通道控制方式 常考

通道具有简单CPU功能、可编程，可管理多个设备同时工作，真正实现CPU与外设并行。

DMA vs 通道：DMA固定传送逻辑，只控制少数同类设备；通道有自己的指令集，可控制多种不同设备，能力更强。

3. 缓冲技术

3.1 缓冲的目的与分类 重点

目的：解决CPU与外设速度不匹配的矛盾，提高CPU与外设之间的并行性，减少对CPU的中断频率。

  • 硬件缓冲：外设自带专用寄存器，侧重微观匹配（Printer、CD-ROM等）
  • 软件缓冲：OS在内存中开辟缓冲区，侧重宏观调度（4种实现方式）

3.2 软件缓冲的4种实现方式 必考

  • 单缓冲：一个缓冲区，外设与CPU操作串行，不能同时读写
  • 双缓冲：两个缓冲区交替使用，实现CPU与外设的并行交换
  • 环形缓冲：多个等大缓冲区链成环形链表，三个指针 Head/Full/Empty 分别跟踪起始/满/空位置
  • 缓冲池：多个等大缓冲区的公共池，系统统一管理分配/回收，分三类队列：空缓冲队列、输入缓冲队列、输出缓冲队列

3.3 SPOOLING（假脱机）技术 必考

SPOOL（Simultaneous Peripheral Operation On Line）：将独占设备改造成共享设备的技术。

原理：截获向独享设备（如打印机）输出的数据，暂存到磁盘上的输入井/输出井（虚拟设备）排队，再由守护进程逐个输出到真实设备。

内存缓冲区用于匹配CPU与磁盘速度（临时性），磁盘井用于持久化排队（SPOOLING基石）。

易错：SPOOLING使独占设备变为逻辑上的共享设备，多个进程可同时"使用"打印机，实际上在排队。

3.4 磁盘缓冲（buffer cache）与sync 常考

Linux将写磁盘数据先放入buffer，不立即写盘（提高效率）。若此时断电，数据丢失。

sync 命令强制flush缓冲区到磁盘。自动写盘时机：① kflush内核线程周期写；② buffer已满。

4. 磁盘设备

4.1 磁盘物理结构 重点

磁盘结构：盘面（face）-- 磁道（track）-- 扇区（sector），相同半径的磁道构成柱面（cylinder）。

扇区是磁盘的基本寻址单位（通常512字节）。访问时间组成：

磁盘访问延迟 = 队列时间 + 控制器时间 + 寻道时间 + 旋转时间 + 传输时间

前两项可忽略；寻道时间（~8ms）和旋转时间（半圈~4ms，5400rpm）是主要瓶颈，优化目标是最小化这两项。

4.2 磁盘调度算法 必考

经典实例：磁头在53，请求队列 98,183,37,122,14,124,65,67

算法原则磁头移动量（实例）缺点
FCFS按请求到达顺序640磁道磁头长途奔袭，忽略途中请求
SSTF优先服务距当前磁头最近的请求236磁道可能饥饿（远端请求长期得不到服务）
SCAN（电梯算法）单向扫到头再折返，顺路服务所有请求236磁道两端延迟不均（靠近折返端的新请求等待长）
C-SCAN单向扫到头后直接跳回另一端重新扫53+199+134磁道两端无请求时仍移动到磁盘端
C-LOOKC-SCAN改进：前面没有请求就回移最优化-- （推荐默认算法）
易错：SSTF不能保证平均寻道时间最短（贪心可能不全局最优），且有饥饿问题；SCAN解决了饥饿但延迟不均；C-LOOK是最合理的默认算法。

4.3 磁盘寻址：CHS与LBA 重点

CHS寻址：地址 = 柱面(C) + 磁头(H) + 扇区(S)，扇区编号 A = c×H×S + h×S + s。CHS最大支持 256×1024×63×512B ≈ 8GB，已淘汰。

LBA（Logical Block Addressing）：将所有扇区线性编号（0到N），外圈扇区比内圈多（等密度）。硬盘控制器固件将LBA转换为物理地址。

硬盘布局：第1扇区为MBR（主引导记录），含引导代码和主分区表（最多4个基本分区）。

计算机自举：ROM自举程序 --> 读MBR到内存 --> MBR引导代码 --> 读引导分区第1扇区 --> 装载OS。

4.4 磁盘访问时间计算 必考

平均旋转延时 = 0.5 / 转速（转/秒）。例：5400rpm，平均旋转延时 = 0.5/（5400/60）= 5.6ms

总访问时间 = 平均寻道时间 + 平均旋转延时 + 传输时间 + 控制器延时

示例：寻道12ms + 旋转5.6ms + 512KB/200MB/s(2.5ms) + 控制器0.2ms = 20.3ms

5. 固态盘SSD

5.1 SSD基本原理 重点

存储介质：NAND Flash，以页（page，4KB）为单位写入，以块（block，512KB，含多页）为单位擦除。

写操作限制：Flash只能把1改为0，不能直接覆盖写。修改时需 read-modify-write：读原块到缓存、修改后写到新页，原页标为无效页。

随机写慢：擦除以块为单位（~1ms），修改一页需复制整块，产生写放大效应（Write Amplification）。

5.2 闪存翻译层FTL 常考

FTL（Flash Translation Layer）由三部分组成：

  • 地址映射（Address Mapping）：将上层文件系统的逻辑扇区号（LBA/LSN）转换为Flash物理页地址 <package,die,plane,block,page>
  • 损耗平衡（Wear Leveling）：让各块的擦写次数均衡分布，延长SSD寿命。分动态（新请求选擦除次数少的块）和静态（定期迁移冷数据）
  • 垃圾回收（Garbage Collection）：定期将有效页搬移到新块，擦除含大量失效页的块，回收存储空间

5.3 SSD与机械磁盘对比 常考
