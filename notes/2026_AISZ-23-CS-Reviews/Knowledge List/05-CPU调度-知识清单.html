一、CPU调度基本概念

什么是CPU调度 必考

CPU调度（进程调度）：操作系统按照一定策略从就绪队列中选择一个进程，将CPU使用权交给该进程。

  • 调度资源：CPU的使用权或时间片
  • 调度对象：进程或线程（原则相同，以进程说明）
  • 进程调度 ⇔ CPU调度

CPU-I/O区间周期 重点

进程执行过程由CPU区间与I/O等待区间交替组成：

  • CPU约束型：以计算为主，CPU区间较多且长（典型：矩阵运算、Matlab）
  • I/O约束型：以I/O为主，有大量短CPU区间（典型：word、ppt）
  • 实验表明：进程中短CPU区间出现频率极高，分布呈指数衰减
易错：CPU区间长度频率分布是"指数衰减"——短区间极多，长区间极少，不是均匀分布。

非抢占式 vs 抢占式调度 必考

类型触发条件使用场景
非抢占式进程主动让出（完成或等待事件）早期多道批处理系统
抢占式时间片到期 / 出现更高优先级进程大多数现代操作系统

二、调度准则

调度评价指标 必考

  • 吞吐量：单位时间完成的任务数量（越大越好）
  • 周转时间 = 完成时刻 - 到达时刻（越小越好）
  • 等待时间：任务在就绪队列中等待的时间总和
  • 响应时间 = 首次运行时刻 - 到达时刻（交互式系统关键指标）
易错：响应时间≠周转时间。响应时间是首次运行到到达的间隔；周转时间是完成到到达的间隔。

调度准则之间的矛盾 重点

  • 响应时间短 --> 前台任务优先级高 --> 后台任务饥饿
  • 吞吐量大 --> 时间片大（切换少） --> 响应时间长
  • 协调多个目标是操作系统复杂性的重要来源

三、调度算法

FCFS — 先到先服务 重点

调度顺序即到达就绪队列的顺序。

示例（5个进程同时到达，CPU区间：P1=10, P2=29, P3=3, P4=7, P5=12）：

  • 执行顺序：P1 P2 P3 P4 P5，完成时刻：10, 39, 42, 49, 61
  • 平均等待时间：(0+10+39+42+49)/5 = 28
  • 平均周转时间：(10+39+42+49+61)/5 = 40.2

特点：公平简单、非抢占；存在护航效应（长作业阻塞后续短作业），不适合交互式系统。

易错：时间片用完后进程变为"就绪态"而非"阻塞态"（阻塞是等待I/O事件）。

SJF — 最短作业优先 必考

每次选择CPU区间最短的进程调度（非抢占版本）。

同上示例用SJF（P3=3, P4=7, P1=10, P5=12, P2=29）：

  • 平均等待时间：(0+3+10+20+32)/5 = 13
  • 平均周转时间：(3+10+20+32+61)/5 = 25.2

SJF是同时到达时平均等待时间最小的调度算法（可证明）。

问题：CPU区间大小必须已知或可预测。

SRJF — 最短剩余作业优先 重点

SJF的可抢占版本：新进程到达时，若其剩余时间 < 当前进程剩余时间，则抢占。

示例（P1到达0需10ms，P3到达5需3ms --> 到时刻5 P1剩余5ms > P3的3ms，P3抢占）：

  • SRJF平均等待时间 = 8.6 < SJF的10.4
易错：SRJF比SJF等待时间更小，但CPU区间仍必须已知。

CPU区间预测：指数平均法 重点

τn+1 = α·tn + (1-α)·τn
其中 tn 是当前CPU区间，τn+1 是下次预测值，α 控制历史与近期的权重。

  • α = 0：忽略近期，关注历史
  • α = 1：只关注当前
  • 通常取 α = 0.5

优先级调度 重点

每个任务关联一个优先权，调度优先权最高的任务。SJF是优先级调度的特例（优先级 = 1/CPU区间）。

饥饿问题：优先级太低的任务一直得不到运行。

解决：老化（Aging）技术——任务优先级随等待时间增长而提升。

易错：FCFS是RR的特例（时间片无穷大时等价FCFS），SJF是优先级调度的特例。这两对关系常考。

RR — 轮询调度 必考

按时间片轮转调度，适合交互式系统。

示例（时间片=10，P1=10, P2=29, P3=3, P4=7, P5=12）：

  • 调度序列：P1 P2 P3 P4 P5 P2 P5 P2，完成：10, 61, 23, 30, 52
  • 平均周转时间：35.2（FCFS=40.2，SJF=25.2）
  • 平均响应时间：16.6（FCFS=28，SJF=13）
  • 上下文切换次数：RR=7次 vs FCFS/SJF=4次

时间片设置原则（折中）：

  • 太大 --> 退化为FCFS，响应时间长
  • 太小 --> 切换开销比例过大（如时间片20ms，切换5ms，则20%开销）
  • 推荐：时间片10-100ms，切换时间约0.1-1ms（1%）
易错：时间片≥最长作业时等同于FCFS。减小时间片不一定减小平均周转时间（示例见教材表格：时间片=3时均值10.75，时间片=6时10.5）。

多级反馈队列 MLFQ 必考

最通用也最复杂的CPU调度算法。

  • 将任务按优先级分为多个队列（如队列0/1/2对应实时/系统/交互/批处理进程）
  • 高优先级队列不空时不调度低优先级队列
  • 任务可在队列间移动：
    • 使用太多CPU时间 --> 降低到低优先级队列
    • 在低优先级等太久 --> 提升到高优先级队列（Aging）

需要决定的参数：队列数目、每队调度算法、升降队列策略、初始分配策略。

彩票调度算法 常考

彩票数表示任务应获得的资源份额。调度器随机选彩票，加载中奖进程运行。

  • 进程A有75张彩票 --> 占75% CPU时间；进程B有25张 --> 占25%
  • 随着时间增加，实际分配比例趋近期望值
  • 优点：简单、支持比例资源分配

调度算法对比总结 必考

算法类型优点缺点
FCFS非抢占公平、简单护航效应，不适合交互式
SJF非抢占最小平均等待时间需知CPU区间，可能饥饿
SRJF抢占比SJF更优需知CPU区间，开销大
优先级抢占/非灵活低优先级饥饿（需Aging）
RR抢占响应时间短，公平上下文切换多，周转时间长
MLFQ抢占最通用，自适应最复杂，参数多
彩票抢占比例分配，简单随机性，短期不精确

四、程序优化基础

优化层次与目标 重点

需要在多个层次优化：算法 --> 数据结构 --> 过程 --> 循环（重点内循环）。

CPE（Cycles Per Element）：处理每个元素的平均时钟周期数，线性模型 T = CPE × n + Overhead。

降低CPE是优化数据密集型循环的关键目标。

普遍有用的优化方法 必考

1. 代码移动（Code Motion）

  • 将循环中每次结果相同的计算移到循环外
  • 示例：将 strlen(s) 从 for 条件中移出，使 lower1（O(n²)）变成 lower2（O(n)）

2. 复杂运算简化（Strength Reduction）

  • 用更简单操作替换昂贵操作：移位/加法代替乘除法
  • 示例：循环中 n*i 改为累加 ni += n，避免每次乘法

3. 公用子表达式共享

  • 将多次计算的相同表达式提取为临时变量

4. 局部变量累积结果（消除不必要内存引用）

  • 用临时/局部变量在循环中累积，最后一次性写回 *dest
  • combine1 --> combine4：CPE从10.12降至1.27（整数加）

妨碍优化的因素 必考

障碍1：函数调用

  • 编译器将函数视为黑盒，在函数附近进行弱优化
  • 原因：函数可能有副作用，或依赖全局状态，对相同参数可能返回不同值
  • 补救：使用 inline 内联函数，或程序员手动代码移动

障碍2：内存别名使用

  • 两个不同的内存引用指向相同位置（C语言允许地址运算，很容易发生）
  • 示例：若 a 和 b 内存区域重叠，修改 b[i] 可能改变 a 的后续元素，编译器不敢优化
  • 补救：引入局部变量在循环中累积，避免重复读写同一内存位置
易错：这两个障碍是编译器无法自动绕过的，必须程序员手动处理。

五、利用指令级并行优化

现代处理器架构 重点

  • 超标量处理器：一个周期执行多条指令，从连续指令流中动态调度
  • 乱序执行：只要不违反数据依赖和控制依赖，处理器可以乱序执行指令
  • 流水线功能单元：将运算分解为多阶段，前阶段值传出后即可开始下一次运算

  • 吞吐量界限：CPE的终极下限 = 1/吞吐量，吞吐量 = 容量/发射时间

循环展开与多累加器 必考

2x1循环展开（x = (x OP d[i]) OP d[i+1]）：

  • 减少循环次数，对整数加法有帮助（达到延迟界限）
  • 其他无改进——括号顺序导致仍然是顺序依赖

2x1a 重组展开（x = x OP (d[i] OP d[i+1])）：

  • 改变括号位置，打破顺序依赖：d[i] OP d[i+1] 无需等待上次结果
  • CPE ≈ 延迟界限/2，接近2倍速度提升
  • 注意：浮点运算缺乏结合性，结果可能不同

2x2 分离累加器（x0 和 x1 独立累积）：

  • 两条独立流水线并行，同样实现CPE ≈ 延迟界限/2
  • 整数加法进一步利用两个加载单元，CPE=0.81

一般 KxU 展开原则：要达到吞吐量界限，需 K ≥ 容量 × 延迟。

易错：循环展开本身（2x1）不改变顺序依赖，必须结合重组（2x1a）或分离累加器（2x2）才能突破延迟界限。

SIMD 向量化 了解

  • SIMD（Single Instruction Multiple Data）：一条指令同时对多个数据执行相同操作
  • AVX2的YMM寄存器：256位，可同时处理 8个32位浮点 或 4个64位浮点
  • 向量化后可比标量最优再提升约4-16倍（Double FP乘法：0.52 --> 0.16 CPE）
