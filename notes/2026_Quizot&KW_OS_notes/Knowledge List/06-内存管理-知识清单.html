1. 背景：程序如何进入内存

1.1 从源程序到进程 必考

程序执行需要经历三个阶段：

  • 编译：源代码 --> 汇编/目标文件，产生逻辑地址（从0开始编址）
  • 链接：合并各目标文件的段，统一分配地址，解析符号引用（静态链接/动态链接）
  • 装入（加载）：loader 将可执行文件装入内存，建立进程地址空间

1.2 重定位的三个时机 必考

  • 编译/链接时重定位：产生绝对代码，只能放在预先确定的位置，不可移动
  • 载入时重定位：装载时修改地址，一旦载入不能再移动
  • 运行时重定位（最常见）：每次访存时由硬件（MMU）动态翻译，支持移动和保护
易错：运行时重定位不是"运行前修改代码"，而是每条访存指令执行时由MMU硬件实时完成地址翻译。

1.3 三类地址概念 重点

  • 逻辑地址（虚拟地址）：程序中使用的地址，从0编址，是段内相对地址
  • 线性地址：段基址 + 逻辑偏移，CPU地址总线可访问的全部地址集合
  • 物理地址：实际物理内存单元的编号，以Byte为单位
  • MMU（Memory Management Unit）：CPU中实现逻辑地址 --> 物理地址映射的硬件电路

1.4 进程交换（Swap） 常考

将阻塞/睡眠进程换出到磁盘，释放内存给其他进程；需要时再换入。

交换要求程序是可重定位的（运行时重定位支持换入后放在不同位置）。

2. 连续内存分配

2.1 固定等长分区 了解

初始化时将内存等分为 k 个分区，大小相同（SecSize）。

  • 优点：算法简单，分配快
  • 缺点：产生大量内部碎片（分配了但未使用的空间）

2.2 固定变长分区 了解

初始化时分成 k 个大小不同的分区，进程选最小满足需求的分区。

  • 仍有内部碎片，灵活性略优于等长分区

2.3 可变分区 重点

根据 reqSize 动态分割内存，数据结构维护空闲分区表和已分配分区表。

reqSize = code + data + stack + heap（stack/heap 需预留空间）

2.4 分区分配算法 必考

算法策略特点
首次适配（First-Fit）找到第一个满足的空闲分区速度快；大分区在前易被切割
最佳适配（Best-Fit）找最小的满足需求的分区搜索整个表，慢；产生大量小碎片
最差适配（Worst-Fit）找最大的空闲分区搜索整个表，慢；剩余分区较大

2.5 碎片问题 必考

  • 外部碎片：空闲空间总量足够但不连续，无法分配给大进程
  • 内部碎片：分区已分配但内部有未使用空间（固定分区产生）
  • 内存紧缩（Compaction）：移动进程，合并外部碎片；时间成本极高（1GB@4B/20ns ≈ 5秒）
易错：外部碎片是可变分区的问题，内部碎片是固定分区的问题；两者不要混淆。

3. 分段内存管理（Segmentation）

3.1 分段思想 重点

程序员视角：程序由若干段组成（主程序段、数据段、栈段、堆段、库段等），每段从0编址。

地址格式：<段号, 段内偏移>

分段将二维地址空间映射到一维物理内存，各段可分散存放。

3.2 段表与地址翻译 必考

每个进程维护一张段表，每条记录包含：段号、基址、长度、保护位。

地址翻译：物理地址 = 段表[段号].基址 + 段内偏移

需检查：偏移 < 段长度（越界检查）；权限检查（读/写/执行）

段号 S 和偏移 O 的计算：S = INT[A/L]，O = A % L（A为逻辑地址，L为段大小）

3.3 分段的优缺点 重点

  • 优点：符合程序员习惯；各段独立编址，便于保护和共享
  • 缺点：产生外部碎片；空间预留问题；内存紧缩代价高

4. 分页内存管理（Paging）

4.1 分页思想 必考

将逻辑地址空间和物理内存都切割成大小相等的片：

  • 页（Page）：逻辑地址空间的固定大小块，典型大小 4KB
  • 页框/页帧（Frame）：物理内存的固定大小块，与页等大

优点：无外部碎片（页框大小固定，物理内存分配不会产生碎片）；内部碎片最多一页大小。

易错：分页消除了外部碎片，但仍有内部碎片（最后一页可能未填满）。

4.2 页表与地址翻译 必考

逻辑地址格式：[页号 | 页内偏移]（页内偏移位数 = log₂(页大小)）

地址翻译：物理地址 = 页表[页号].页框号 × 页大小 + 页内偏移

页表项（PTE）结构：页框号（ppn）、有效位（valid）、保护位（R/W/X）、修改位（dirty）、访问位（accessed）

翻译示例：逻辑地址 0x240，页大小 4K，页号=0，偏移=0x240；查页表得页框号5，物理地址=0x5240

4.3 多级页表 必考

问题：32位地址 + 4KB页 = 2²⁰个页表项，每进程需 4MB 页表，10进程需 40MB。

解决：引入多级页表，顶层页目录常驻内存，未使用的逻辑地址不建立页表项。

32位两级页表地址格式：[页目录号(10位) | 页号(10位) | 页内偏移(12位)]

访存次数：1级页表需2次，2级需3次，n级需n+1次访存。

示例（代码+数据+栈各4M，共12M）：页目录 1 页(4K) + 3个页表各 1 页(4K) = 4K+12K = 16K

易错：多级页表节省的是页表本身占用的内存空间，而非访存次数——访存次数反而增多了。

4.4 TLB（Translation Look-aside Buffer） 必考

TLB 是一组关联快速寄存器（硬件缓存），缓存最近使用的页表项，避免每次都访问内存中的页表。

有效访问时间（EAT）公式：

EAT = HitRate × (TLB时间 + 1次内存访问) + (1-HitRate) × (TLB时间 + 2次内存访问)

示例：TLB=20ns，内存=100ns

  • 命中率80%：EAT = 0.8×120 + 0.2×220 = 140ns
  • 命中率98%：EAT = 0.98×120 + 0.02×220 = 122ns

TLB 容量通常 64~1024 条，命中率高的原因：程序访存具有局部性（时间局部性+空间局部性）。

进程切换时 TLB 需要 flush（或使用 ASID 标记避免 flush）。

4.5 分页的优缺点 重点

  • 优点：靠近硬件，高效利用内存，无外部碎片，易于自动化（硬件实现）
  • 缺点：不符合程序员思考习惯；页表本身占用内存

5. 段页结合内存管理

5.1 段页结合思想 重点

让段面向用户（便于编程、保护、共享），让页面向硬件（消除外部碎片，高效分配）。

地址翻译流程：逻辑地址 --> 段表（得线性地址）--> 页表（得物理地址）

5.2 Intel x86 内存管理 常考

  • 实模式：地址空间1MB，段寄存器存放段基址，逻辑地址=物理地址；物理地址=段寄存器×16+偏移
  • 保护模式：地址空间4GB，段寄存器存放段选择符（索引GDT/LDT），支持段页式管理
  • GDT：全局段描述符表，系统启动时由内核生成，全局共享
  • LDT：局部段描述符表，进程加载时创建，各进程独有
  • CR3寄存器：保存当前进程页目录表的物理地址，进程切换时更新

5.3 段页结合地址翻译步骤 必考

  1. 用段选择符查 GDT/LDT 得到段描述符（基址、限长、权限）
  2. 基址 + 逻辑偏移 --> 线性地址，进行越界和权限检查
  3. 用线性地址的高位查页目录，再查页表，得物理页框号
  4. 物理页框号 + 页内偏移 --> 物理地址

完整过程需查段表+页目录+页表，效率低；TLB 缓存可大幅提升性能。

5.4 内存管理三核心词 必考

翻译（逻辑地址 --> 物理地址）、保护（越界检查、权限检查）、内存分配（如何管理空闲空间）
