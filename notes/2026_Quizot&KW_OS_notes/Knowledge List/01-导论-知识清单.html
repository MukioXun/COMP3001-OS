本章整体概览

本章核心问题：操作系统是什么、为什么需要它、它如何被实现。后续章节的进程、内存、文件系统都是对"OS是硬件与用户程序之间的中介"这一核心思想的具体展开。

前置知识补充

计算机系统的层次结构：硬件(ISA) → OS → 系统软件 → 应用程序。每层是下层的抽象，上层看不到下层实现细节。

程序从源码到执行：hello.c → 预处理(.i) → 编译(.s) → 汇编(.o) → 链接(可执行) → 加载进虚拟内存 → CPU执行。OS参与了加载、执行、I/O每一步。

逐模块深度讲解

模块一：操作系统是什么 必考

定义：OS是一个大型程序系统，负责软硬件资源分配；控制和协调并发活动；提供用户接口。

为什么需要OS：硬件不能被应用直接安全高效地使用。OS本质是"硬件的管理者"和"应用程序的服务者"。

OS提供的三大抽象：

  • 进程：对处理器、主存和I/O设备的抽象，让每个程序以为自己独占CPU。
  • 虚拟内存：对主存和磁盘I/O设备的抽象，让每个程序以为自己有独立的大内存空间。
  • 文件：对I/O设备的抽象，统一了磁盘、键盘、网络的访问接口。
易错：进程是对"处理器+主存+I/O设备"三者的抽象，不只是CPU；虚拟内存是对"主存+磁盘"的抽象，不只是内存。这是经典选择题考点。

模块二：操作系统的四大特征 必考

1. 并发（Concurrence）：两个或多个事件在同一时间间隔内发生（宏观并行，微观交替）。区别并行（Parallelism）：并行是同一时刻真正同时发生，需要多核。单核CPU上只有并发，没有并行。并发是单处理机OS最重要的特征。

2. 共享（Sharing）：系统资源可供多个并发进程共同使用。分两类：互斥共享（临界资源，如打印机，同一时刻只允许一个进程）；同时访问（分时复用如单核CPU，或真正并发如只读文件）。共享与并发互为依存。

3. 异步性（Asynchronism）：任务以不可预知的速度推进，结果具有不确定性。根因是多进程竞争资源。进程可能在任意指令后被时钟中断剥夺CPU；无同步机制时，共享变量的结果取决于指令交错顺序，每次运行可能不同。这是后续"并发与同步"章节要解决的核心问题。

4. 虚拟（Virtual）：把一个物理实体变成若干个逻辑上的对应物。虚拟CPU（每进程以为独占CPU）、虚拟内存（每进程以为有独立地址空间）都是虚拟化体现，是OS最核心的关键技术之一。

易错：并发是"时间段内"，并行是"同一时刻"。单核CPU上程序A和B"并发"是微观轮流占CPU，不是同时执行。

模块三：操作系统发展历史 重点

主线：每代OS的出现都为了解决上一代效率瓶颈。

  • 手工操作：CPU速度提升后，人工操作时间远大于机器运行时间，CPU大量空等。
  • 批处理系统：联机批处理直接连主机；脱机批处理通过磁带中转，避免主机等待低速设备。
  • 多道程序设计：内存同时放多个程序，一个等I/O时CPU切给另一个，大幅提升CPU利用率。这是现代OS的基础。
  • 分时系统：多用户共享主机，每用户感觉独占，引入时间片轮转调度。

Unix历史要点：1969年Ken Thompson用汇编写出Unix原型（为运行星际旅行游戏）；1970年设计B语言；1972年Dennis Ritchie设计C语言；1973年用C重写Unix。

模块四：操作系统内核结构 必考

为什么讨论内核结构：OS内核代码庞大，哪些放内核态、哪些放用户态直接影响性能、安全性和可维护性。

宏内核（Monolithic Kernel）：所有OS功能（进程/内存/文件系统/驱动）都在内核空间同一地址空间，通过函数调用相互访问。优：性能高，调用开销小。缺：一个服务崩溃可能导致整个系统崩溃，不易扩展。代表：Linux、BSD。

微内核（Microkernel）：仅将基本IPC、内存管理、调度放内核空间；文件系统、驱动、网络作为独立用户态进程，通过IPC通信。优：单服务崩溃不影响全局，可扩展，更安全。缺：IPC开销大，性能低于宏内核。代表：QNX、Minix、HURD。

宏内核微内核
服务运行位置内核态（同一地址空间）用户态（不同地址空间）
通信方式函数调用IPC
执行速度快慢
安全性低（一崩全崩）高（服务隔离）
可扩展性差好
代表Linux, BSD, WindowsQNX, Minix, HURD
易错：微内核"内核本身尺寸小"，但总代码量（含用户态服务）反而更大。不要混淆"内核尺寸"和"代码量"。

模块五：系统调用 必考

为什么需要系统调用：用户程序在用户态运行，无法直接访问硬件资源。OS提供一组受控接口让用户程序请求内核服务，同时保证安全隔离。

执行流程：用户程序调用库函数(如printf) → 库函数设置系统调用号(eax=1) → syscall/int 0x80陷入内核态 → 保存所有寄存器 → 中断处理程序按调用号分发到内核函数(如sys_write) → 执行完毕 → 恢复寄存器 → 返回用户态。

五大系统调用类型： 进程控制(fork/exit/wait)； 文件管理(open/read/write/close)； 设备管理(ioctl)； 信息维护(getpid/alarm)； 通信(pipe/shmget/mmap)。

易错：系统调用不是函数调用。函数调用不切换特权级；系统调用必须通过syscall/int指令从用户态陷入内核态，有显著开销。

模块六：系统启动流程 重点

上电 → ROM固件执行 → bootstrap loader定位内核 → 内核加载到内存 → 内核初始化启动。GRUB是常见引导程序，支持从多磁盘、多版本选择内核。引导可分两阶段：ROM代码加载boot block，再由boot block加载完整bootstrap loader。

常见误区与易混淆点

易错：进程是对"处理器+主存+I/O"三者的抽象，不只是CPU。
易错：并发≠并行。单处理机OS有并发，无并行（微观上同一时刻只执行一条指令）。
易错：微内核"内核尺寸小"，但IPC接口代码量大，总代码量反而更大。
易错：系统调用涉及特权级切换，开销远大于普通函数调用。
易错：存储器层次结构最顶层（速度最快）是寄存器，不是高速缓存。

本章总结

OS是硬件与应用之间的"中介层"，通过并发管理、资源共享、虚拟化和统一接口（系统调用）让多个程序安全高效地共用一套硬件。内核结构（宏/微/混合）决定了中介层的实现方式和性能取舍。
